CHAPTER ONE · ORIGIN

為什麼自建

主公裝了 Obsidian、看一眼、覺得太複雜。500 個外掛、Graph View 一團亂、永遠都是檔案。

但「第二大腦」這個概念太重要、不能放棄——人的腦子裝不下、靈光一現的想法每天蒸發、 看到的好東西轉頭就忘。所以拍板自建——12 小時從零到上線、不裝插件、不學語法、 用我們最熟悉的工具：純 HTML + Cloudflare Pages + Claude + Gemini，做一個只服務一個人的腦。

不是 Obsidian 的替代品。是換一條路、走得更深。

CHAPTER THREE · HOW IT WORKS

使用說明

五個面向：投入（讓靈光不蒸發）→ 處理（AI 自動整理）→ 整理（DIKW 三階）→ 共腦（七員一起長大）→ 視覺化（看見全貌）→ 共享（送出去）。

投入端 · 讓靈光不再蒸發

  • 投入頁：貼網址 / 寫想法 / 丟連結 / 拖截圖 / 上傳 PDF。
  • LINE 自動投入：在 LINE 直接傳網址、或寫 30 字以上想法 → 自動進 brain。
  • 瀏覽器外掛：右鍵任何網頁「投進笙之道」、一秒入庫。
  • PWA iPhone 分享：Add to Home Screen 後、IG / 任何 app 分享 → brain。
  • Agent 推回：七員工作時看到值得記的、自動推回 brain inbox。

處理端 · AI 自動整理

  • 投入後 AI 立即摘要 + 自動標籤 + 開場白（17 角色視角擇一）。
  • YouTube 連結：Gemini 直吃影片、看畫面 + 聽聲音、產出時間軸摘要。
  • IG / TikTok / Threads 短影音：cobalt.tools 下載 + Gemini Files 分析。
  • 一般網址：fetch 內文 + Claude Sonnet 摘要、開啟 prompt caching。
  • 截圖 / PDF：Vision 讀內容、自動 OCR + 摘要。

整理端 · DIKW 三階

  • 來水（Inbox）：剛投入、待主公決定。Data → Information。
  • 儲藏（Archive）：已整理永久、全文搜尋 + 語意搜尋 + 標籤雲 + 過月回顧。Information → Knowledge。
  • 悟（Sanctum）：思維模型 / 鐵律 / Skill 三類、主公親手萃取的智慧。Knowledge → Wisdom。

共腦端 · 七員一起長大

  • Sanctum 內容自動同步到本機 memory/。
  • 七員（韓信／郭嘉／墨子／蕭何／小喬／周瑜／傳令）session 啟動讀 memory → 自動繼承主公的新模型。
  • 不用改任何 SKILL.md、agent 自然變聰明。

視覺化端 · Atlas 思維地圖

  • D3.js 力導向節點圖、看 Sanctum 全貌。
  • 節點顏色分三類（思維模型 / 鐵律 / Skill）、邊厚度看共同 tag。
  • 太極自轉感、慢動效、不滿屏。
  • Daily 每日思維卡：每天抽一張、配 AI 引言、可跟該模型對話。

共享端 · 送禮分享

  • 點任一張 sanctum 卡 →「送這張卡」。
  • 三種輸出：Markdown / HTML / 圖片（gpt-image-1 自動排版）。
  • 三種有效期：24 小時 / 7 天 / 永久。
  • 可加密碼保護、可看點擊數。

CHAPTER FIVE · PHILOSOPHY

笙之道核心

上善若水。水善利萬物而不爭、
處眾人之所惡、故幾於道。
  • DIKW 三階提煉：資料 → 資訊 → 知識 → 智慧、自動推進、不靠人工分類。
  • 萃取過才共享：只有智慧層（Sanctum）該被七員繼承、原始材料留在 archive 不汙染腦。
  • 開放式腦：能跟任何系統互動、雙向迴流、不被綁定。資料隨時可匯出。
  • 道 ⊃ 團隊 ⊃ 工具：笙之道是道、若水聯盟是團隊、Skill 是執行——三層分明、不混淆。

不堆功能、堆連結。不收資料、收意義。
