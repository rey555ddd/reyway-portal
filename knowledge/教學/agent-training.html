為什麼要用 AI Agent？

在學怎麼用之前，先搞清楚「為什麼這套系統跟你以前用過的完全不一樣」。

Ch.1
基礎概念

Claude Code 是什麼？

很多人把 Claude Code 當成「更聰明的 ChatGPT」，這是最大的誤解。

💬 一般 Claude（對話版）

  • 你問，它答，僅此而已
  • 不能操作你的電腦
  • 不能讀寫你的檔案
  • 關掉視窗就完全忘記
  • 每次都要重新介紹自己

⚡ Claude Code（有手有腳）

  • 可以寫程式並直接跑起來
  • 可以讀寫你電腦上的任何檔案
  • 可以搜尋網路、查資料
  • 可以幫你操作瀏覽器
  • 透過 Memory 永久記住你
  • 透過 Skill 具備專業知識

重要觀念：Claude Code 不是「聊天工具」，它是可以接受派工、自主完成任務的 AI 員工。你說「幫我分析這份廣告數據並寫成週報」，它就真的去做完，而不是只告訴你「你可以這樣做」。

它能做到的事（實際例子）

📊 數據分析

讀取廣告報表 CSV，自動分析 ROAS 趨勢，輸出圖表與建議

🌐 建工具網站

從零打造電商數據站、廣告分析儀表板，部署到網路上

📝 生成文件

自動產出會議記錄、週報、合意終止書、企劃書

🤖 自動化流程

定期跑廣告報告、自動整理數據、發送摘要

🔍 研究調查

搜尋競品、整理市場資料、分析消費者評論

🎨 素材生成

AI 生圖、寫文案、製作廣告素材，整合品牌知識

🎯 第一步：先知道目的

在使用 AI 之前，最重要的事是釐清最終目標。不是「我想用 AI 做廣告」，而是「我想讓廣告週報從手動 4 小時變成自動 10 分鐘」。目的越清楚，AI 的輸出越精準。

🔄 最後一步：存回記憶

每次你覺得某個指令特別好用、某個工作流程跑得很順——把它存進 Memory。下一次 AI 就從這個優化後的版本出發，而不是從零開始。這就是「越用越聰明」的秘密。

實際例子（廣告投手）：第一次叫 AI 分析廣告，花了很多時間調整格式。把這個好版本存進 Memory 後，下一次 AI 一開口就用對格式，一次到位。三個月後，這位投手的廣告分析 SOP 全都住在 AI 的 Memory 裡。

⚡ 笙闆心法：數字指令

飛輪要跑得快，就要把「常用動作」變成肌肉記憶。
笙闆把每個關鍵動作壓縮成一個數字——打一個數字 = 啟動一整套流程。這是實戰用下來，個人覺得最實用的心法。

⚠️
這些指令需要你自己輸入到對話框

第一次用 AI 時，它還不認識這些數字——所以你要先用完整句子讓 AI 記住規則，之後它就會懂。
捷徑：第一次先複製下面這段「規則設定」貼給 AI，它就會接受所有數字指令了。

📋 點開複製：第一次教 AI 數字指令的提示詞 ↓ 展開

把這段整個複製貼給 AI，它就會記住數字指令的對應動作：

從現在起請記住以下數字指令對應的動作，
之後我打數字（可多個用空格隔開）你就依序執行：

0 = 查我的收件夾、重載最新規則、回報目前進行中／卡住的任務
1 = 工作日誌：整理本次對話的所有決策、產出、待辦，存進 Memory
2 = 我這邊做完了（驗證、輸入密碼、檔案放好），你接著做下一步
3 = 完整品質審查：用嚴格主管 + 陌生使用者 + 功能實測三個視角審查並直接修
4 = 升級 Skill：把本次的踩坑解法、穩定 pattern、新發現補進對應 Skill 檔案
5 = 全力跑：細節你自己決定，不用每步問我，最後給結果就好
9 = 重載工作習慣：重新讀取所有共用規則檔案

請回覆「已記住」並列出你理解的對應，我之後就會用這些數字了。

💡 進階：記得後可以再加一句「請把這套規則寫進你的 Memory，下次新對話也記得」——這樣以後每次新對話都能直接打數字。

Ch.3
架構

Agent — 有角色的 AI

Agent 不是一個更聰明的機器人，而是一個有職稱、有職責、有工作規則的 AI 員工。

一般 AI vs Agent 的差別

一般 AI = 任何人問什麼，它答什麼，沒有立場
Agent = 有角色設定、知道自己的職責範圍、照著公司規則做事

就像「一個路人」vs「一個你公司的員工」——員工知道公司文化、知道自己負責什麼、知道什麼事要回報給誰。

指揮鏈 — 誰找誰說話

笙闆只跟韓信、郭嘉、周瑜說話。墨子、蕭何、小喬的工作由韓信發包，笙闆不直接派工給他們。

這就像你不會直接叫工廠員工做事，你說給你的助理，助理再去協調——這樣才不會亂。

一個任務怎麼跑完的

笙闆說：「做一個廣告投放分析工具」
↓ 韓信接收，拆解任務，派工給周瑜（行銷規格）+ 墨子（前端 UI）+ 蕭何（Meta API 串接）
↓ 三人各自去做，完成後回報韓信
↓ 韓信驗收（重點），挑問題請對方修，過關才整合成品
↓ 韓信回報笙闆，笙闆說「好」或「調一下這裡」
↓ 完成，結果存進 Memory

🧬 進階心法：用 AI 寫提示詞給 AI

這是把 Agent Team 玩到極致的關鍵——最好的提示詞，不是你寫的，是 AI 寫的。
因為人腦寫提示詞容易抽象、漏細節；AI 寫提示詞會非常精準、完整、可執行。

❌ 自己寫提示詞

「幫我做一個廣告分析工具」

  • 太抽象、AI 自己腦補
  • 漏掉重要細節（資料來源、欄位、格式）
  • 來回修改 5-6 次才對
✅ AI 寫提示詞

先跟 AI 討論需求，請它產出完整提示詞，再貼到新對話。

  • 提示詞涵蓋所有細節
  • 新對話一啟動就精準動作
  • 一次到位，不用反覆修

▸ 笙闆的標準操作流程

1
開「綜合對話」

這是你的總指揮所。所有新任務都先在這個對話裡跟 AI 討論，把需求講清楚。

2
請 AI「產出一個完整的提示詞」

討論完之後，跟 AI 說：

「請根據剛剛討論，幫我寫一個完整的提示詞，
包含背景、目標、步驟、產出格式、注意事項，
讓另一個 AI 看到就能直接動手。」
3
開新對話，貼上提示詞執行

把 AI 寫好的提示詞複製，貼到一個新對話視窗。這個新對話會一啟動就精準動作，不用浪費時間互相摸索。

為什麼這就是「Agent 派工 Agent」的核心

這個流程的本質是：綜合對話的 AI = PM，新對話的 AI = 執行員。

PM 比較懂全局、會問對的問題、會把需求拆得清楚——所以由 PM AI 寫提示詞，給執行員 AI 執行。
這就跟真實公司一樣：老闆不會直接寫 SOP 給工程師，老闆告訴 PM，PM 把規格寫清楚再給工程師。

這就是為什麼需要 PM——不只是分配工作，更是把模糊的人類需求翻譯成精準的執行規格。

🛡️ PM 驗收：減少你被打擾的次數

派工出去之後，不要讓每個 Agent 直接交給你看——應該先回報給 PM，由 PM 驗收、挑問題、請對方修，過關之後才整合好交給你。

Ch.4
長期記憶

Memory — 讓 AI 記住你

為什麼 ChatGPT 每次都要重新介紹自己？因為它沒有 Memory。笙闆 AI的 AI 員工，每次開工就自動讀取你留下的所有記錄。

😵 沒有 Memory 的世界

  • 每次開新對話，AI 都像新員工
  • 你的偏好、決策、工作方式要重新解釋
  • 同樣的錯誤一犯再犯
  • 累積不了任何「公司知識」

✅ 有 Memory 的世界

  • 每次開工，AI 自動讀上次的決策記錄
  • 知道你的工作習慣與偏好
  • 記住公司的每條規則
  • 你優化的結果永久保存

Memory 的本質

Memory 不是什麼神奇技術，它就是放在電腦裡的 Markdown 文字檔案。AI 開工前讀這些檔案，就像員工上班前看昨天的交接記錄。你可以新增、修改任何一條。

memory/
  ├── 全員共用/   ← 所有 AI 員工啟動時必讀（公司鐵律）
  ├── 部門/
  │   ├── 軍師/   ← 郭嘉的知識庫（策略/人資/財務）
  │   ├── 工匠/   ← 墨子的知識庫（前端技術）
  │   └── 謀士/   ← 周瑜的知識庫（行銷方法論）

  │   └── 像素辦公室/
  └── 個人日誌/   ← 每個 AI 員工自己維護的日誌

什麼東西該存進 Memory？

📋 決策記錄

這次討論完的結論、定案的方向、拍板的選項。下次 AI 直接從這個結論繼續，不用重新討論。

⚙️ 工作習慣

你喜歡報告怎麼格式、指令習慣怎麼下、哪些字眼不要用。存進去，AI 永遠記住。

🏢 公司鐵律

不能說什麼、一定要做什麼、對外文案的標準。這些全存進全員共用資料夾，每個 AI 都知道。

✨ 最佳實踐

某個工作流程跑得特別順？某個提示詞特別好用？存進去，下次直接複用。

🐛 踩過的坑

某個做法行不通、某個平台有限制？記錄下來，AI 下次自動避開，不重複犯錯。

📊 品牌知識

產品規格、受眾定位、競品分析、行銷策略。讓 AI 一開口就像品牌老手。

飛輪效應：你每次優化後把結果存進 Memory，AI 下次起點就更高。做了三個月，你的 AI 員工對你公司的了解，會比任何一個剛到職的人類員工都要深。這就是護城河。

Ch.5

Skill — 讓 AI 一開口就是老手

Ch.6
指令技巧

如何給 AI 好的指令

AI 的輸出品質，80% 取決於你怎麼下指令。這章是笙闆用了兩年整理出來的心法。

核心原則：好的指令包含四個要素——目的（我要達成什麼）+ 背景（現在的狀況是什麼）+ 限制（有哪些條件限制）+ 格式（輸出要長什麼樣子）

壞指令 vs 好指令

❌ 模糊指令

「幫我分析廣告」

✅ 好指令

「分析這份 Meta 廣告週報（CSV 附件），找出 ROAS 最差的三組受眾，每組給我一個優化建議。輸出格式：三個重點 + 各一句建議，貼給投手看的。」

❌ 模糊指令

「幫我寫文案」

✅ 好指令

「幫我寫一支家用清潔產品的 IG 貼文，目標是讓第一句話就讓媽媽想停下來看。語氣輕鬆，不要用廣告腔。結尾放一個問句跟受眾互動。300字以內。」

❌ 模糊指令

「整理一下我們的人事狀況」

✅ 好指令

「讀取 memory 裡的合意終止試算記錄，整理一份給人資看的操作說明：什麼時候談、說什麼、要準備哪些文件。用條列式，語氣平靜不帶評判。」

兩個常用指令範例

用兩種典型情境示範——數據分析型跟方法論導入型。看完你就知道怎麼套用到自己的工作。

📊

財務 / 人事 / 月報（數據分析型）
月報分析指令
讀取這份損益表（[附上檔案]）， 找出本月最大的成本異常， 跟上月比較後給我三個重點， 用主管能直接看的格式。

📌
看出共通公式了嗎？「載入什麼 Skill」+「讀取什麼資料」+「要做什麼」+「輸出什麼格式」——這四件事講清楚，AI 一次就到位。
Ch.6.5
把工具調順

權限 × 模型 — 讓 Claude Code 跑更順

下指令的方式對了，還要把工具本身調到順——權限模式決定 AI 多大膽，模型選擇決定 AI 多聰明。兩個調好，效率再 ×2。

🔓 一、權限模式：別讓 AI 一直問你

Claude Code 預設每次要動檔案、跑指令都會問你「可以嗎？」——一個任務問你 30 次，你的注意力都被切碎了。
解法：用快捷鍵 ⇧ ⌘ M（Shift + Command + M）切換權限模式。

四種權限模式
Ask permissions
1

每件事都問你。新手安全模式，但會被打斷很多次。

Accept edits
2

改檔案不問，跑指令還是會問。適合單純改檔。

Plan mode
3

AI 只規劃不動手。大任務先讓它寫計畫，你看完再放行。

笙闆預設
Bypass permissions
4 ✓

全部不問，AI 自己做主。日常主力模式，省掉 90% 打擾。

▸ 笙闆的權限策略

日常

用 Bypass permissions（4 號）——AI 全自動跑，你看最終成品就好。

大改

重大架構改動先切到 Plan mode（3 號）——AI 先寫計畫給你看，確認後再動手。

敏感

處理財務、客戶資料、上線到正式機 → 切回 Ask permissions（1 號）穩當點。

▸ 進階：一句話授權

不想切換模式？對話開頭加一句話，等於臨時授權：

# 全任務最高授權
這個任務我給你最高權限，
不用每步問我，細節你自己決定，
有重大判斷再回報，最後給我成品就好。
# 局部授權
接下來這 30 分鐘全力跑，
檔案、部署、設定都自己處理，
做完打 1 給我工作日誌。
💡
小技巧：數字指令「5」就是濃縮版的「最高權限」——打 5 = 全力跑、細節不用問。

Ch.7
最高效率心法

多對話 × Sub Agent — 平行作業

真正會用 AI 的人，不是「跟一個 AI 聊很久」——而是同時開好幾個對話、每個處理不同主題、每個都是有專業的 Agent。這才是 AI 主管的工作節奏。

最高效率公式
每個對話 = 一個 Agent + Sub Agent 平行作業 + 同時推進多個專案 = 一個人開一間公司

一、多對話：每個對話一個主題，平行進行

不要把所有事擠到同一個對話裡。換主題 = 換對話，這樣每個對話都很乾淨、AI 不會混亂、你也好管理。

對話 #1
📊 行銷：廣告週報

載入廣告分析 Skill，分析本週 Meta 廣告數據。

對話 #2
📝 營運：人事文件

載入文件範本 Skill，產出個人化的合約文件。

對話 #3
🎨 設計：IG 視覺

載入品牌設計規範，產出 5 張下週貼文圖。

⏱ 三件事同時跑，不是排隊等

二、Sub Agent：把任務「樹狀分派」

Sub Agent 的概念是：AI 有能力同時開好幾個「分身」分頭做事。
你叫它派 3 個 Sub，速度就 ×3；而每個 Sub 又可以再派出 3-5 個自己的 Sub——這就是一棵 Agent 樹，速度可以指數放大。

# 第一層：派 3 個 Sub Agent
請同時派三個 Sub Agent 處理皮爾卡登首頁：
- Sub A：設計 Hero 視覺
- Sub B：寫品牌文案
- Sub C：切版上 RWD

# 第二層：給每個 Sub「再分派」的權限
每個 Sub 都可以再派出 3 個自己的 Sub Agent 平行作業，
最後由你（主 Agent）整合所有結果回報給我。
💡
關鍵心法：你不用管每個 Sub 怎麼分工——你只要給 Sub 主管「再派人」的權限，AI 會自己決定要拆幾個。你只看最終整合結果。這就是「當 AI 主管」的實戰技巧。

三、最高效率畫面：一個人開一間公司

想像你的螢幕現在開著三個對話：

  • 📊 對話 1：周瑜 在跑廣告週報（同時派了 2 個 Sub Agent 抓 Meta 數據 + 寫摘要）
  • 🎨 對話 2：小喬 在做下週的 IG 視覺（同時派了 3 個 Sub Agent 出 3 種風格）
  • 🏗️ 對話 3：蕭何 在串電商後台 API（同時派了 2 個 Sub Agent 寫測試 + 寫文件）

這個畫面的意思是：你一個人，等於同時驅動 3 個對話 × 平均 2.5 個 Sub Agent ≈ 7-8 個 AI 員工同時工作。
這就是「能擔任 AI Agent 主管」的價值。

對話太長怎麼辦？

📝 任務做完了 → 打 1

  • AI 整理本次決策 + 產出
  • 存進 Memory，下個對話接著用
  • 關掉這個對話，開新的

🔄 任務還沒完，但對話太長

  • 輸入 /compact
  • AI 自動壓縮對話歷史
  • 釋放空間，繼續在同個對話
💡
笙闆觀察：很多人卡在「跟一個 AI 講很久」、然後抱怨 AI 變笨。真正的解法是分對話——每個對話一個目標、做完打 1 收尾、下次重新開乾淨的。三個月後，你會發現自己一天的產出，是過去一週的量。

Ch.9
安全使用

注意事項

AI 是強大的工具，但它不是萬能的，也不是絕對正確的。這幾條鐵律一定要記住。

⚠️
AI 會犯錯，而且它不會主動說「我不確定」。數字、日期、法規、人名——這類資訊一定要人工確認，不能直接使用。

🔒 絕對不給 AI 的資料

  • 任何帳號密碼
  • 信用卡號碼或銀行帳戶
  • 員工身分證字號
  • 客戶完整個資（含聯絡方式）
  • 公司未公開的財務數字

✅ 給 AI 之前先想一下

  • 這份資料可以讓外部人看嗎？
  • 如果這段對話被截圖出去，有問題嗎？
  • AI 的輸出要對外發布之前，有沒有人工確認過？
  • 敏感決策最終要由人來拍板，不能完全靠 AI 判斷

AI 工作鐵律速查

  • 結論先說，10 秒讓主公看懂，再說細節
  • 不說技術術語（API / deploy / commit 一律換成白話）
  • AI 能做的事全部自己做，不要反覆問主公
  • 生成類工具必須有：一鍵範例 + 微調對話框 + 意見回報按鈕
  • 文案一律用 Claude + 行銷 Skill；圖片一律走 fal.ai
  • 任務結束前一定打「1」存工作日誌
