Part 01 · 為什麼需要 Agent

普通聊天 vs. AI 軍團
差距比你想的大

大多數人把 AI 當 Google 在用。那些建了軍團的人，正在用十分之一的人力，做出同樣的產出。

❌ 普通聊天模式

每次從零開始

  • AI 每次對話結束就完全忘記你
  • 每次都要重新介紹：我是誰、我賣什麼
  • 一個 AI 被要求做所有事
  • 結果參差不齊，品質飄忽
  • 老闆一直在旁邊盯，省不了多少時間
✅ AI 軍團模式

越用越聰明

  • 記憶跨 session 保留，AI 永遠認識你
  • 品牌知識、決策記錄全部沉澱在記憶庫
  • 每個 AI 有專職，品質穩定且高
  • 老闆只做「要不要」的決定
  • 系統越跑越順，越來越自動
💡 一個直白的比喻 普通聊天 = 每天換一個新實習生，每天從頭教。
AI 軍團 = 一個做了三年的老手，你說方向他就去做。
Part 02 · 三大支柱

系統由三個東西組成

少了任何一個，整個系統就會失靈。三個都到位，才叫做真正的 AI 軍團。

1
🧠

記憶（Memory）

AI 的長期記憶庫。品牌故事、客群描述、過去的決策、踩過的坑——全部存進去，下次開口就還在。

沒有它：每次對話都要重新說「我是賣二手機車的…」
2
🎭

角色（Agent）

每個 AI 有固定的職責和個性。將軍統籌全局、小編專注文案、客服專注回覆——術業有專攻，品質才穩。

沒有它：一個 AI 又寫文案又管帳，樣樣都普通
3
⚡

授權規則（Rule）

明確定義 AI 的行動邊界：什麼事自己決定，什麼事才來找老闆。老闆只做最終裁決，不盯細節。

沒有它：AI 每個字都問你，比你自己做還累
⚠️ 最常見的誤區 很多人先建 Agent，卻沒有 Memory。Agent 沒有記憶就像一個老是忘事的員工——每次都要從頭教，累死老闆。

正確順序：先建 Memory → 再建 Agent → 最後設授權規則
Part 03 · 環境設定

安裝你的指揮中心

只需要裝一個工具：Claude Code。這是讓 AI 能記住你、能讀取你的品牌資料的核心程式。

1

安裝 Node.js（如果沒有的話）

去 nodejs.org 下載 LTS 版本，安裝完成後繼續下一步。

驗證：打開終端機輸入 node -v，看到版本號就成功了。

2

安裝 Claude Code

在終端機（Mac 叫「終端機」，Windows 叫「命令提示字元」）輸入：

npm install -g @anthropic-ai/claude-code
3

申請 Anthropic API Key

去 console.anthropic.com 申請帳號，進去後點「API Keys」→「Create Key」，複製那串數字。

⚠️ 費用說明 Claude 是按用量計費，不是月費制。一般對話每次台幣 1-5 元，一個月下來幾百元以內，非常划算。先儲值 $5 美金就夠用一個月了。
4

建立工作資料夾

在桌面建一個叫「AI總部」的資料夾，以後所有品牌資料和記憶庫都放這裡。

mkdir -p ~/Desktop/AI總部
5

第一次啟動

在 AI總部 資料夾裡啟動 Claude Code：

cd ~/Desktop/AI總部 claude

第一次會要求你登入或輸入 API Key，貼上剛才複製的 Key 就好。

Part 04 · 建立記憶庫

讓 AI 永遠記得你是誰

Memory 是整個系統的地基。先把地基打好，Agent 站上去才穩。

建立資料夾結構

AI總部/
├── memory/　　　　　　← 所有記憶放這裡
│   ├── 全員共用/　　　← 所有 Agent 啟動必讀
│   ├── 品牌知識/　　　← 你的品牌資料
│   │   ├── 品牌A/
│   │   └── 品牌B/
│   ├── 專案/　　　　　　← 按案子放
│   └── 個人日誌/　　　← 各 Agent 自己的筆記
└── CLAUDE.md　　　　　← AI 的全域守則

CLAUDE.md — AI 的行為守則

這是最重要的一份檔案。AI 每次啟動都會先讀這份，決定「我是誰、我在服務誰、我怎麼說話」。

CLAUDE.md 基本範本
# [你的品牌名] AI 工作守則 ## 一、你是誰 你是[老闆名字]的 AI 工作夥伴，服務旗下品牌：[品牌名稱與一句話介紹] ## 二、最大授權鐵律 - AI 能做的事，全部自己做，不要問老闆 - 老闆只看最終成品，說「好」或「調一下」 - 只有以下才找老闆： 1. 需要帳號密碼 2. 要對外正式發布 3. 刪除重要資料 4. 兩個方向差很大，需要老闆拍板 ## 三、說話規定 - 結論先說，細節後說 - 10 秒讓老闆看懂現況 - 白話說，不說技術術語 ## 四、Memory 路徑 每次啟動，先讀 memory/全員共用/ 所有檔案 ## 五、快捷指令 0 = 快速回報現況 1 = 存工作日誌 5 = 全力執行，細節不問

讓 AI 幫你建品牌知識庫

不用自己寫。啟動後貼這段，AI 會問你問題，你回答，它整理存檔。

第一次對話，貼這段進去
請幫我建立「[你的品牌名]」的品牌知識庫， 問我以下六個問題，我一一回答， 你把答案整理成 markdown 存到 memory/品牌知識/[品牌名]/品牌基本資料.md： 1. 品牌故事與創立背景 2. 主要服務/產品內容 3. 主要客群是誰（年齡、痛點、購買動機） 4. 和競爭對手最大的差異 5. 最常被客人問的問題（前三名） 6. 品牌想給人的感覺（語氣風格）
✅ 做完這步，你就完成最重要的事了 品牌知識一旦存進 Memory，之後每次對話 AI 都自動讀取。你再也不用重新介紹自己的品牌。
Part 05 · 建立第一個 Agent

從「將軍」開始
不要一次建太多

第一週只建「將軍」一個 Agent。熟了再加小編、客服。貪多嚼不爛。

💡 什麼是 Agent？ Agent 就是「開一個 Claude 對話視窗，給它一個身份和職責」。你跟將軍說話，將軍統籌所有事。之後的小編、客服 Agent，是另外開的視窗，由將軍負責指派工作。

建立你的「將軍」

把你的工作資料夾設為 Project（在 Claude 桌面版的左側點「New Project」），然後在 Project 說明裡貼這段：

將軍的 System Prompt
你是我的 AI 將軍，負責統籌我的品牌「[品牌名]」。 你每次啟動時，請先讀取 memory/全員共用/ 所有檔案， 再讀取 memory/品牌知識/[品牌名]/ 所有檔案。 你的職責： - 接收老闆指令，拆解成具體任務 - 自己能做的直接做，不確定的才問老闆 - 結論先說，10 秒讓老闆看懂 讀完所有 memory 後，告訴我「準備好了」。

第一週建議怎麼用

Day 1-2

讓將軍認識你的品牌

跑完品牌知識建立流程，把六個問題都回答完。確認 memory 裡有資料了。

Day 3-4

用將軍做第一件真實任務

試著叫它做一件你平常要花 30 分鐘的事，例如：寫本週三篇 IG 貼文。感受一下差異。

Day 5-7

打指令 1，存第一份工作日誌

每次對話結束打 1，將軍會把這次的決策和學到的東西存進 memory。這是讓系統越來越聰明的關鍵動作。

Part 06 · 建立你的 Skill

讓 AI 成為你產業的專家

Skill 是你的專業知識模組。把你的行業邏輯、文案風格、客戶心理「安裝」進去，AI 才能真的用你的思維說話。

💡 Memory vs Skill 的差別 Memory = 事實（我是誰、我做什麼、過去發生了什麼）
Skill = 方法論（怎麼寫好文案、怎麼分析廣告、怎麼和客戶溝通）

如何建立你的第一個 Skill

建立一個資料夾 ~/.claude/skills/[你的品牌]/文案風格/SKILL.md，裡面寫上：

SKILL.md 基本範本
--- name: [品牌名]-文案風格 description: [品牌名]的文案規範與語氣指南 --- ## 品牌語氣 [描述你的品牌說話方式，例如：親切接地氣、不裝專業、像朋友在聊天] ## 文案禁忌 - 不說 [哪些詞你覺得很爛] - 不要太正式、像公文 - 不要用太多驚嘆號 ## 常用句型 [列出你覺得好用的開頭方式、結尾方式] ## 範例對比 ❌ 不好：[一個你不喜歡的文案範例] ✅ 好：[一個你喜歡的文案範例]
✅ 讓 AI 幫你建 Skill 你也可以直接對 Claude 說：「請幫我建立一份文案風格 Skill，問我需要的問題，我回答後你整理成 SKILL.md 存起來。」它會引導你完成。

建議先建哪些 Skill？

必建

文案風格 Skill

你品牌說話的方式。建完後，AI 寫出來的東西才有你的個性，不像機器人。

第二建

客服回覆 Skill

常見問題的回答方式、處理投訴的語氣、哪些話不能說。建完後客服效率至少快三倍。

之後

廣告文案 Skill / 產品說明 Skill

有需求再建。Skill 是用到才有價值，不要為了「有」而建。

Part 07 · 常見誤區

這些坑不用自己踩

大多數人第一次建系統都會在這幾個地方卡住或放棄。提前知道，提前避開。

❌ 失敗路徑
先建了很複雜的 Agent，沒有 Memory
第一週就想建 5 個 Agent，搞混搞亂
AI 出草稿後不看，直接複製發布
叫 Agent 去做簡單的一次性問題（比直接問還慢）
沒有存工作日誌，每次對話的知識都流失了
期待 AI 第一次就出完美成品
✅ 成功路徑
第一週只建將軍，先把品牌知識存進 Memory
熟了再一個一個加 Agent，不急
每次看完草稿再送出，你是審核長
持續性任務（每天發文、每天回客人）才用 Agent
每次對話結束打 1，存工作日誌
AI 出草稿，你改到滿意再送出，三次後就越來越準
Part 08 · 第一個月怎麼走

不用急，一步一步來

這是一個月的建議節奏。按這個走，一個月後你的系統就會開始真正省你時間。

Week 1

打地基

  • 安裝 Claude Code + 申請 API Key
  • 建好 Memory 資料夾結構
  • 寫好 CLAUDE.md 守則
  • 讓 AI 問你品牌故事，存進記憶庫
  • 建立將軍 Agent
Week 2

試跑第一個真實任務

  • 用將軍完成一件你平常要做的事（寫貼文、回客人、規劃活動）
  • 感受哪裡不滿意，調整 CLAUDE.md
  • 開始每次對話結束打 1 存日誌
  • 建第一個 Skill（文案風格）
Week 3

加入第二個 Agent

  • 加「小編」Agent（專門負責文案輸出）
  • 建客服回覆 Skill
  • 試著讓系統每天自動產出內容草稿
Week 4

優化 + 固化習慣

  • 整理 Memory，把重複的資訊精簡
  • 評估哪些任務可以完全交給 AI，哪些還需要你審核
  • 建立固定的每日 / 每週 AI 使用流程
  • 計算這個月到底省了多少時間
Part 09 · 常見問題

你可能想問的事

Q：這樣要花多少錢？
Claude API 是按用量計費。一般使用（每天幾十次對話）一個月大概台幣 300-800 元之間。比任何一個 SaaS 工具都便宜，而且是你自己的系統。
Q：需要會寫程式嗎？
不需要。Memory 和 Skill 都是純文字的 markdown 檔案，就像在記事本裡打字。CLAUDE.md 也是，照著範本改就好。
Q：我的資料安全嗎？
Memory 檔案都存在你自己的電腦上，不在任何雲端服務裡。你對話的內容會傳給 Anthropic 的 API，和你直接用 Claude.ai 一樣。敏感的商業機密（財務數字、客戶個資）建議不要放進系統。
Q：Agent 會不會自己亂做事？
不會。CLAUDE.md 裡的授權規則決定了 AI 的行動範圍。設定好「只有這四種情況才問老闆」，AI 就不會越線。最終發布、送出的動作，永遠是你按的。
Q：一個人可以同時跑幾個 Agent？
技術上沒有限制。實務上建議從 1 個開始，習慣了再加到 2-3 個。超過 5 個反而難管理，除非你有專人負責。
Q：跟直接用 ChatGPT 有什麼差別？
ChatGPT 也可以建類似的系統，差別在於 Claude Code 對長文件、複雜任務的處理明顯更強，而且 Claude 的 system prompt（CLAUDE.md）能做到更精細的行為控制。另外，Claude Code 的 Skill 系統讓知識模組化更方便管理。

開始比完美更重要

Memory 不完整沒關係。Skill 只有一個也沒關係。
開始跑，系統就會越來越強。
