Claude 三刀流

Cowork 做事、Code 建網站、Claude.ai 思考——三刀各司其職，根據任務性質互相交棒。省時間、省 token、省錢。

Cowork
Claude Code
Claude.ai

流程

怎麼交棒

看任務性質決定誰來做，然後把結果交給下一個。

① Cowork
整合一切，
執行複雜操作

開始的地方。在桌面直接討論專案、拆解需求。不只停在文字——Cowork 能整合電腦、瀏覽器、所有工具，立刻動手執行：整理檔案、呼叫 API、設定環境變數、自動部署。

確認需求規格 設定 Cloudflare 環境變數 觸發部署 / 驗證結果 整理桌面檔案
② Claude Code
建網站、做事、
實現想法

Cowork 講清楚要做什麼之後，丟給 Code 去實作。Code 在終端直接生成代碼、建網站、修 bug、跑測試——不用人在旁邊盯，極速完成。

建立 HTML / JS 頁面 寫 Cloudflare Pages Functions 修 bug / 重構 Push 到 GitHub 自動部署
③ Claude.ai
思考、記憶、
深度對話

長期記憶最好的地方。Cowork 建立的 Skill（品牌知識庫、合規規範、行銷方法論）都能讀取。需要深度思考、策略推演、文案發想的時候，切來這邊——帶著完整背景知識對話，效率最高。

調用品牌 Skill 寫文案 策略長篇討論 合規審查 大量問答 / 推演
↩ 回 Cowork
遇到複雜執行任務，
再交回 Cowork

討論完新方向、或需要操作電腦執行任務，再切回 Cowork。這是個循環，不是線性流程——每個工具只做它最擅長的事，剩下交出去。

執行新一輪部署 自動化任務 操作桌面應用 下一個專案討論
最複雜的事先用 Cowork 談清楚、然後丟給 Code 去蓋——蓋好的結果再用 Claude.ai 加上品牌 Skill 往深了推。三個工具各守自己的邊界，才不會浪費 token 做重工。
—— 笙闆（陳睿笙） · 淨淨 CleanClean 創辦人

對比

只用其中一個，會很吃虧

Cowork 單獨用 — 幾個複雜專案就會爆流量。
Code 單獨用 — 不容易累積Skill、只能建站也很多瑣碎流程。
Claude.ai 單獨用 — 只能出張嘴，無法幫你做。

對比

三個工具，各有各的長短

不是讓你選哪個，而是告訴你什麼時候用哪個。

工具 優點 缺點 定位
Cowork • 能驅動電腦和瀏覽器
• 整合一切系統工具
• 自動執行複雜操作
• 最適合實際做事 		• Session 流量有限

• 做網站比 Code 慢 		整合系統，實際執行的樞紐

💡 Cowork 模型選擇小貼士

使用 Cowork 時要記得切換模型。非必要的話不要開啟 Sonnet 以上，大部分的對話其實用基本款（Haiku）就可以了。

簡單任務（整理檔案、設定環境變數）→ Haiku
中等複雜（寫簡單代碼、自動化）→ Sonnet
超級複雜（深度推理、策略規劃）→ Opus

這樣既能省錢，又不會浪費 token。

最終好處

為什麼交叉使用更有效率

在 Cowork 裡把品牌知識、行銷方法論、產品規格、合規規範都建成 Skill——之後在 Claude.ai 和 Code 任何對話都能隨時直接調用，不用每次重新解釋背景。

三個好處

交叉使用的意義不只是工作流順暢，而是用最少的成本和精力，完成最複雜的事。

1. 長期記憶 — Claude.ai 記著所有 Skill 和背景知識，不用重複解釋
2. 複雜任務委派 — Cowork 自動執行，Cowork 用基本款就夠，省 token 又省事
3. 網站快又便宜 — Code 建網站極快，成本遠低於其他方案

結論：既能省錢，又能省事。
